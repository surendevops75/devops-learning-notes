# SonarQube Configuration

## What is SonarQube Configuration?

**SonarQube Configuration** allows Jenkins to connect to a SonarQube server and perform **static code analysis** as part of the CI/CD pipeline.

Instead of manually configuring the SonarQube server in every pipeline, Jenkins administrators configure it once globally.

SonarQube helps teams identify:

- Bugs
- Vulnerabilities
- Code Smells
- Duplicated Code
- Maintainability Issues
- Security Hotspots

This enables developers to detect issues early before code reaches production.

---

# Why Do We Need SonarQube Configuration?

A project may compile successfully but still contain:

- Security vulnerabilities
- Poor coding practices
- Memory leaks
- Duplicate code
- Maintainability issues

Compilation checks only whether code builds successfully.

SonarQube checks the **quality of the code**.

## Hardcoding Token

Wrong

```groovy

```

Use Jenkins Credentials.

---

## Skipping Quality Gate

Wrong Pipeline

```text

↓

Sonar Scan

↓

Deploy
```

Recommended

```text
Build

↓

Sonar Scan

↓

Quality Gate

↓

Deploy
```

---

## Wrong Server URL

Example

```text
http://localhost:9000
```

Use the production SonarQube URL instead.

---

## Ignoring Scan Results

Running scans without reviewing issues reduces the value of static analysis.

Fix critical findings before deployment.

---

# Best Practices

- Configure SonarQube globally.
- Store authentication tokens in Jenkins Credentials.
- Enforce Quality Gates in all production pipelines.
- Run unit tests before SonarQube analysis to include code coverage.
- Block deployments if the Quality Gate fails.
- Regularly update SonarQube and scanner plugins.

---

# Troubleshooting

## Cannot Connect to SonarQube

Verify

- Server URL
- Network connectivity
- Firewall rules

---

## Authentication Failed

Check

```text
Manage Jenkins

↓

Credentials
```

Verify the configured SonarQube token.

---

## Quality Gate Never Returns

Possible causes

- SonarQube server overloaded
- Background tasks still running
- Webhook not configured

Verify SonarQube background task status and webhook configuration.

---

## Analysis Failed

Check

- Project key
- Project name
- Scanner configuration
- Java version
- Maven configuration

# Email Configuration

## What is Email Configuration?

**Email Configuration** allows Jenkins to send email notifications for important pipeline events such as:

- Build Success
- Build Failure
- Deployment Status
- Test Results
- Security Scan Reports
- Approval Notifications

Instead of manually sending emails, Jenkins automatically notifies developers, QA teams, and operations teams.

---

# Why Do We Need Email Configuration?

Imagine a production deployment fails at midnight.

Without email notifications,

no one knows the deployment failed until someone manually checks Jenkins.

Email notifications provide immediate visibility into pipeline events.

# Best Practices

- Store SMTP credentials in Jenkins Credentials.
- Use the Email Extension Plugin for production pipelines.
- Notify only relevant teams.
- Send HTML emails with useful build information.
- Include links to Jenkins build logs instead of large attachments.
- Limit notifications to important pipeline events.
