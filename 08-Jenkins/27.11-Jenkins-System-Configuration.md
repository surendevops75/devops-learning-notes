# Best Practices

Following best practices helps keep Jenkins stable, secure, scalable, and easier to maintain in production environments.

---

# 1. Configure Tools Globally

Configure commonly used tools such as JDK, Maven, Docker, NodeJS, Terraform, and SonarQube from **Global Tool Configuration** instead of hardcoding paths.

```text
Manage Jenkins

↓

Global Tool Configuration

↓

Configure Once

↓

Reuse Everywhere
```

Benefits

- Easier maintenance
- Consistent builds
- Cleaner Jenkinsfiles

---

# 2. Never Store Secrets in Pipelines

Never write passwords, tokens, or API keys inside Jenkinsfiles.

Wrong

```groovy
password = "admin123"
```

Correct

```text
Manage Jenkins

↓

Credentials

↓

Secure Storage
```

Use Jenkins Credentials for:

- GitHub Tokens
- AWS Credentials
- Docker Registry Credentials
- SonarQube Tokens
- Slack Tokens
- SSH Keys

---

# 3. Use Dynamic Build Agents

Do not execute builds on the Jenkins controller.

Recommended

```text
Controller

↓

Provision Agent

↓

Run Pipeline

↓

Delete Agent
```

Benefits

- Better performance
- Improved scalability
- Enhanced security

---

# 4. Standardize Tool Versions

Ensure all agents use the same versions of:

- JDK
- Maven
- Docker
- NodeJS
- Terraform

Example

```text
Agent A

JDK 21

↓

Agent B

JDK 21

↓

Agent C

JDK 21
```

This avoids inconsistent build behavior.

---

# 5. Keep Plugins Updated

Regularly update Jenkins plugins to receive:

- Security patches
- Bug fixes
- New features
- Compatibility improvements

Avoid unnecessary plugins that increase maintenance and attack surface.

---

# 6. Enforce Quality Gates

Do not deploy code that fails quality checks.

```text
Build

↓

Unit Tests

↓

SonarQube

↓

Quality Gate

↓

Pass?

↓

Yes

↓

Deploy

────────────

No

↓

Stop Pipeline
```

---

# 7. Scan Artifacts Before Deployment

Security scanning should be part of every production pipeline.

Example

```text
Docker Build

↓

Trivy Scan

↓

High Severity?

↓

Yes

↓

Fail Pipeline

────────────

No

↓

Push Image
```

Infrastructure should also be scanned.

```text
Terraform Code

↓

Checkov Scan

↓

terraform apply
```

---

# 8. Use Least-Privilege Access

Grant Jenkins only the permissions it actually requires.

Example

```text
Jenkins

↓

AWS IAM Role

↓

Limited Permissions

↓

Deploy Only
```

Avoid using administrator privileges.

---

# 9. Enable Backup and Recovery

Regularly back up:

- Jenkins Home
- Job Configurations
- Plugins
- Credentials
- Shared Libraries

Example

```text
Daily Backup

↓

Remote Storage

↓

Disaster Recovery
```

---

# 10. Monitor Jenkins Health

Continuously monitor:

- CPU
- Memory
- Disk Usage
- Build Queue
- Failed Jobs
- Agent Status

Example

```text
Jenkins

↓

Prometheus

↓

Grafana

↓

Alerts
```

---

# 11. Use Shared Libraries

Avoid duplicating pipeline logic.

```text
Shared Library

↓

Reusable Functions

↓

Multiple Pipelines
```

Benefits

- Reusability
- Easier maintenance
- Standardized pipelines

---

# 12. Secure Jenkins Access

Implement proper access control.

Recommended

- RBAC
- SSO
- Multi-Factor Authentication (if supported)
- HTTPS
- Audit Logging

```text
Users

↓

Authentication

↓

Authorization

↓

Jenkins
```

---

# 13. Separate Environments

Use separate pipelines or folders for:

- Development
- Testing
- Staging
- Production

```text
GitHub

↓

Development

↓

Testing

↓

Staging

↓

Production
```

This reduces deployment risks.

---

# 14. Automate Infrastructure

Provision infrastructure using Terraform instead of manual creation.

```text
Git

↓

Terraform

↓

AWS

↓

Infrastructure Ready
```

Benefits

- Version control
- Repeatability
- Reduced configuration drift

---

# 15. Maintain Clean Workspaces

Remove unnecessary files after pipeline execution.

```groovy
post {
    always {
        cleanWs()
    }
}
```

Benefits

- Saves disk space
- Prevents build contamination
- Improves agent performance

# Key Takeaways

- Centralize tool and credential management to simplify administration.
- Enforce security checks such as SonarQube, Trivy, and Checkov before deployment.
- Use dynamic build agents and standardized tool versions for scalable, consistent builds.
- Protect Jenkins with RBAC, least-privilege access, HTTPS, and regular backups.
- Monitor Jenkins continuously and keep the environment updated to maintain a reliable production CI/CD platform.

# Troubleshooting

Even with proper configuration, Jenkins administrators encounter issues related to tools, credentials, plugins, agents, networking, and cloud integrations.

A structured troubleshooting approach helps quickly identify and resolve problems.

---

# Standard Troubleshooting Workflow

```text
Pipeline Failed

↓

Identify Failed Stage

↓

Read Console Output

↓

Check Jenkins Logs

↓

Verify Configuration

↓

Fix Issue

↓

Re-run Pipeline
```

Always start with the **Console Output**, as it usually contains the exact error.

---

# 1. Tool Not Found

## Problem

```text
mvn

command not found
```

or

```text
java

command not found
```

or

```text
terraform

command not found
```

### Possible Causes

- Tool not installed
- Incorrect tool configuration
- Wrong tool name in Jenkinsfile
- PATH not configured

### Solution

Verify

```text
Manage Jenkins

↓

Global Tool Configuration
```

Check tool installation

```bash
java -version
mvn -version
docker --version
node -v
terraform version
```

---

# 2. Git Checkout Failed

## Problem

```text
Authentication Failed
```

or

```text
Repository not found
```

### Possible Causes

- Invalid Git credentials
- Wrong repository URL
- Missing access permissions
- Expired Personal Access Token

### Solution

Verify

```text
Manage Jenkins

↓

Credentials
```

Also verify

- Repository URL
- Branch name
- Git plugin configuration

---

# 3. Docker Permission Denied

## Problem

```text
permission denied

/var/run/docker.sock
```

### Possible Causes

- Jenkins user not in Docker group
- Docker service stopped

### Solution

Verify

```bash
groups jenkins
```

If necessary

```bash
sudo usermod -aG docker jenkins
```

Restart Jenkins.

---

# 4. SonarQube Connection Failed

## Problem

```text
Unable to reach SonarQube Server
```

### Possible Causes

- Server unavailable
- Wrong URL
- Invalid token
- Firewall restrictions

### Solution

Verify

```text
Manage Jenkins

↓

System

↓

SonarQube Servers
```

Test

```bash
curl http://sonarqube.company.com:9000
```

# Key Takeaways

- Always begin troubleshooting by reviewing the Jenkins Console Output and identifying the stage where the pipeline failed.
- Verify global tool configurations, credentials, plugins, and agent connectivity before investigating application code.
- Cloud-based issues often involve IAM permissions, network connectivity, or agent provisioning failures.
- Use a systematic approach to isolate infrastructure, configuration, or application-related problems.
- A consistent troubleshooting workflow minimizes downtime and accelerates CI/CD pipeline recovery.
