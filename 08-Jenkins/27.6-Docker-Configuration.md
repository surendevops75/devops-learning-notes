# Docker Configuration

## What is Docker Configuration?

**Docker Configuration** allows Jenkins to use Docker for building, tagging, pushing, and running container images as part of CI/CD pipelines.

Instead of manually configuring Docker inside every pipeline, administrators configure Docker once on Jenkins controllers or build agents.

Docker is one of the most important tools in modern DevOps because applications are packaged into portable containers before deployment.

---

# Why Do We Need Docker Configuration?

Modern applications are deployed as containers.

A typical deployment flow is:

```text
Source Code

↓

Compile

↓

Build Docker Image

↓

Push Image

↓

Deploy Container
```

Without Docker,

Jenkins cannot:

- Build container images
- Push images to registries
- Run containerized tests
- Deploy containerized applications

# Where to Configure?

Docker is **not configured directly inside Jenkins** like Maven or JDK.

Instead,

Docker must be installed on the Jenkins Controller or Build Agent.

Example

```text
Ubuntu Server

↓

Install Docker

↓

Verify Docker

↓

Add Jenkins User

↓

Restart Jenkins
```

# Give Jenkins Access to Docker

The Jenkins user must belong to the Docker group.

Example

```bash
sudo usermod -aG docker jenkins
```

Restart Jenkins after updating group membership.

---

# Verify Jenkins Access

Login as Jenkins user

```bash
docker ps
```

Expected

```text
CONTAINER ID

IMAGE

STATUS
```

If permission is denied,

verify Docker group membership.

# Common Mistakes

## Docker Not Installed

Pipeline

```text
docker

command not found
```

Install Docker on the build agent.

---

## Jenkins User Cannot Access Docker

Example

```text
permission denied

/var/run/docker.sock
```

Solution

Add Jenkins user to Docker group.

---

## Docker Service Not Running

Verify

```bash
systemctl status docker
```

Start service

```bash
sudo systemctl start docker
```

---

## Hardcoding Registry Credentials

Wrong

```bash
docker login -u admin -p password
```

Correct

Use Jenkins Credentials.

---

# Best Practices

- Install Docker on dedicated build agents instead of the Jenkins controller.
- Use Jenkins Credentials for registry authentication.
- Scan images with **Trivy** before pushing to a registry.
- Use versioned image tags instead of `latest`.
- Regularly update Docker Engine to supported versions.
- Remove unused images to reclaim disk space.

---

# Troubleshooting

## Docker Command Not Found

Run

```bash
docker --version
```

Install Docker if missing.

---

## Permission Denied

Verify

```bash
groups jenkins
```

Ensure the Jenkins user belongs to the Docker group.

---

## Docker Build Failed

Check

- Dockerfile syntax
- Build context
- Base image availability
- Disk space

---

## Push Failed

Verify

- Registry credentials
- Network connectivity
- Repository permissions
- Image tag

---

# Key Points

- Docker Configuration enables Jenkins to build, run, and push container images.
- Docker must be installed on the Jenkins controller or, preferably, on build agents.
- The Jenkins user requires permission to access the Docker daemon.
- Container images should be scanned for vulnerabilities before being pushed to a registry.
- Docker is a core component of modern CI/CD pipelines targeting Kubernetes platforms such as Amazon EKS.
