## Scenario 61

### Interview Question

A Jenkins pipeline starts failing because GitHub API rate limits have been exceeded. How would you troubleshoot it?

### Production-Level Answer

GitHub API rate limiting usually occurs due to excessive API requests, unauthenticated access, or multiple Jenkins jobs sharing the same credentials.

### Approach

Check:

- GitHub API response
- Rate limit headers
- Authentication method
- Number of webhook events
- Poll SCM configuration
- Concurrent jobs

Verify remaining API quota.

### Common Root Causes

- Poll SCM enabled
- Unauthenticated API calls
- Multiple jobs sharing one PAT
- Frequent branch indexing

### Best Practices

- Use Webhooks instead of Poll SCM
- Authenticate all API requests
- Reduce unnecessary API calls
- Monitor API usage

## Scenario 70

### Interview Question

A Jenkins deployment fails because Terraform detects infrastructure drift. What would you do?

### Production-Level Answer

Infrastructure drift means the actual infrastructure differs from the Terraform state. Identify the source of the drift before applying changes.

### Approach

Check:

- Terraform plan
- State file
- Recent manual changes
- Cloud console
- Git history

Determine whether the drift is expected or unauthorized.

### Common Root Causes

- Manual infrastructure changes
- Failed Terraform Apply
- State inconsistency
- Resource modification outside Terraform

### Best Practices

- Enforce Infrastructure as Code
- Avoid manual cloud changes
- Review Terraform plans
- Detect drift regularly
- Maintain controlled change management
