## Scenario 61

### Interview Question

A Jenkins pipeline starts failing because GitHub API rate limits have been exceeded. How would you troubleshoot it?

### Production-Level Answer

GitHub API rate limiting usually occurs due to excessive API requests, unauthenticated access, or multiple Jenkins jobs sharing the same credentials.

### Approach

Check:

- GitHub API response
- Rate limit headers
- Authentication method
- Number of webhook events
- Poll SCM configuration
- Concurrent jobs

Verify remaining API quota.

### Common Root Causes

- Poll SCM enabled
- Unauthenticated API calls
- Multiple jobs sharing one PAT
- Frequent branch indexing

### Best Practices

- Use Webhooks instead of Poll SCM
- Authenticate all API requests
- Reduce unnecessary API calls
- Monitor API usage

### Interview Question

A Jenkins pipeline suddenly starts failing across every project after infrastructure maintenance. What would be your troubleshooting strategy?

### Production-Level Answer

When multiple pipelines fail simultaneously, the problem is almost always shared infrastructure rather than individual applications.

Focus first on common dependencies before debugging each pipeline separately.

### Approach

Verify:

- Jenkins Controller
- Build Agents
- Git server
- Artifact repository
- Kubernetes cluster
- Docker daemon
- AWS services
- Network
- DNS
- IAM
- Storage
- Monitoring dashboards

Identify the first shared component that became unavailable.

### Common Root Causes

- Network outage
- DNS failure
- Shared storage issue
- Git server outage
- Kubernetes cluster issue
- Artifact repository unavailable
- Cloud service disruption
- Expired shared credentials

### Best Practices

- Always investigate shared infrastructure first
- Use centralized monitoring
- Build dependency health dashboards
- Configure infrastructure redundancy
- Perform RCA after every major incident
