# Jenkins Artifacts

## Introduction

A **Jenkins Artifact** is any file generated during a build that needs to be preserved after the pipeline completes.

Artifacts are typically the final outputs of the build process, such as JAR files, WAR files, Docker image metadata, test reports, coverage reports, or deployment packages.

Instead of rebuilding the application every time, downstream jobs and deployment pipelines can reuse these archived artifacts.

---

# Why Do We Need Artifacts?

Without artifacts

```text
Developer Push

↓

Build

↓

JAR Created

↓

Pipeline Ends

↓

JAR Deleted
```

Problems

- Cannot deploy later
- Cannot download build output
- Cannot promote the same build
- Must rebuild every time

---

With artifacts

```text
Developer Push

↓

Build

↓

Generate Artifact

↓

Archive Artifact

↓

Deploy Multiple Times
```

Benefits

- Reusable builds
- Faster deployments
- Build traceability
- Easier rollback
- Immutable releases

# Artifact vs Source Code

| Source Code | Artifact |
|-------------|----------|
| Written by developers | Generated by build |
| Stored in Git | Stored in Jenkins or Artifact Repository |
| Frequently changes | Immutable after build |
| Editable | Should not be modified |

# Archive Artifacts

Jenkins provides the `archiveArtifacts` step to preserve build outputs.

Example

```groovy
archiveArtifacts artifacts: 'target/*.jar'
```

This stores all JAR files from the `target` directory.

---

# Archive Multiple Files

Example

```groovy
archiveArtifacts artifacts: '''
target/*.jar,
reports/**/*.html,
dependency-check-report/**
'''
```

---

# Fingerprinting Artifacts

Artifact fingerprinting uniquely identifies files across builds.

Benefits

- Track artifact usage
- Verify artifact integrity
- Trace deployments
- Identify downstream consumers

Example

```groovy
archiveArtifacts(
    artifacts: 'target/*.jar',
    fingerprint: true
)
```

# Artifact Storage

Artifacts may be stored in

```text

↓

Archived Artifacts

------------------------

Amazon S3

------------------------

JFrog Artifactory

------------------------

Sonatype Nexus Repository

------------------------

Azure Artifacts
```

Large organizations typically use a dedicated artifact repository instead of storing artifacts only in Jenkins.

---

# Jenkins vs Artifact Repository

| Jenkins Archive | Artifact Repository |
|-----------------|--------------------|
| Build outputs | Long-term storage |
| Limited retention | Enterprise retention |
| Basic download | Version management |
| Pipeline usage | Organization-wide sharing |
| Small projects | Large production environments |
