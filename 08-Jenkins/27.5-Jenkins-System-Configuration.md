# JDK Configuration

## What is JDK Configuration?

**JDK Configuration** allows Jenkins administrators to configure one or more Java Development Kit (JDK) installations that can be used by pipelines and build jobs.

Many applications such as:

- Java
- Spring Boot
- Maven
- Gradle
- SonarQube Scanner

require a JDK to compile and execute Java code.

Instead of specifying the Java installation path in every pipeline, Jenkins provides centralized JDK management.

---

# Why Do We Need JDK Configuration?

A Java application cannot be compiled without a JDK.

Consider a Spring Boot application.

```text
Developer

↓

GitHub

↓

Jenkins

↓

Compile Java

↓

Build JAR

↓

Docker Image

↓

Deploy
```

If no JDK is available,

the build stops immediately.

# Example Jenkinsfile

```groovy
pipeline {

    agent any

    tools {

        jdk 'JDK-21'

    }

    stages {

        stage('Verify Java') {

            steps {

                sh '''
                java -version
                javac -version
                '''
            }

        }

    }

}
```

Jenkins automatically sets

```text
JAVA_HOME
```

and updates the system

```text
PATH
```

for the pipeline.

# Common Mistakes

## Java Not Installed

Pipeline

```text
Java Not Found
```

Verify

```bash
java -version
```

---

## Wrong JAVA_HOME

Incorrect

```text
/usr/java
```

Correct

```text
/usr/lib/jvm/java-21-openjdk
```

Verify

```bash
echo $JAVA_HOME
```

---

## Different JDK Versions

```text
Agent A

JDK-17

----------------------

Agent B

JDK-21

----------------------

Agent C

JDK-11
```

This can produce inconsistent build results.

Standardize versions wherever possible.

---

## Hardcoding JAVA_HOME

Avoid

```bash
export JAVA_HOME=/usr/lib/jvm/java-21
```

Instead,

use Jenkins Global Tool Configuration.

---

# Best Practices

- Configure JDK globally instead of hardcoding paths.
- Use meaningful names such as **JDK-17** and **JDK-21**.
- Standardize Java versions across build agents.
- Install JDK manually in production environments.
- Test applications before upgrading Java versions.
- Remove unused JDK versions periodically.

---

# Troubleshooting

## Java Command Not Found

Run

```bash
java -version
```

Install Java if missing.

---

## Compilation Failed

Verify

- JAVA_HOME
- JDK installation
- Jenkins Tool Configuration
- Maven or Gradle configuration

---

## Wrong Java Version

Run

```bash
java -version
```

Ensure the Jenkins pipeline is loading the correct JDK.

---

## Tool Not Found

Check

```text
Manage Jenkins

↓

Tools

↓

JDK Installations
```

Verify that the tool name matches the Jenkinsfile.

# Key Points

- JDK Configuration allows Jenkins to centrally manage Java installations.
- Pipelines reference JDK names instead of hardcoded installation paths.
- Enterprise environments commonly maintain multiple JDK versions for different applications.
- Jenkins automatically configures `JAVA_HOME` and updates the `PATH` for the selected JDK.
- Standardizing JDK versions across build agents ensures consistent and reliable builds.
