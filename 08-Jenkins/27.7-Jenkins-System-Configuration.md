# NodeJS Configuration

## What is NodeJS Configuration?

**NodeJS Configuration** allows Jenkins administrators to configure one or more Node.js installations that can be used by Jenkins pipelines and jobs.

Node.js is widely used for developing:

- React applications
- Angular applications
- Vue applications
- Express.js APIs
- NestJS applications
- Frontend build tools

Instead of hardcoding the Node.js installation path in every pipeline, Jenkins provides centralized Node.js management.

---

# Why Do We Need NodeJS Configuration?

Modern web applications require Node.js for:

- Installing packages
- Building frontend applications
- Running unit tests
- Executing npm scripts
- Creating production builds

Without Node.js,

Jenkins cannot build JavaScript-based applications.

## NodeJS Not Installed

```text

```

Example

```text

↓

```

## Different Versions on Different Agents

```text
Agent A

NodeJS 18

--------------------

Agent B

NodeJS 20

--------------------

Agent C

NodeJS 22
```

Different versions may produce different build results.

Standardize Node.js versions.

---

## Hardcoding Node Path

Avoid

```bash
/opt/node/bin/npm install
```

Use Jenkins Tool Configuration instead.

---

# Best Practices

- Configure Node.js globally.
- Use LTS versions in production.
- Standardize Node.js versions across all agents.
- Install Node.js manually in enterprise environments.
- Keep `package-lock.json` under version control.
- Periodically update Node.js after compatibility testing.

# Terraform Configuration

## What is Terraform Configuration?

**Terraform Configuration** allows Jenkins administrators to configure a Terraform installation that can be used by Jenkins pipelines to provision and manage infrastructure.

Terraform is one of the most widely used **Infrastructure as Code (IaC)** tools. It enables teams to create, update, and destroy cloud infrastructure using configuration files instead of manually creating resources.

Instead of hardcoding the Terraform executable path inside every pipeline, Jenkins provides centralized Terraform management.

---

# Why Do We Need Terraform Configuration?

Modern infrastructure is managed using Infrastructure as Code.

Instead of manually creating AWS resources,

DevOps teams use Terraform to provision:

- VPC
- EC2
- EKS
- RDS
- IAM
- ALB
- Route53
- S3

Without Terraform,

Infrastructure provisioning becomes manual and error-prone.

# Best Practices

- Configure Terraform globally.
- Standardize Terraform versions across all agents.
- Use remote state storage (such as Amazon S3) for production.
- Protect state files with proper access controls.
- Scan Terraform code with **Checkov** before applying changes.
- Include manual approval before `terraform apply` in production.
- Store cloud credentials securely using Jenkins Credentials.
