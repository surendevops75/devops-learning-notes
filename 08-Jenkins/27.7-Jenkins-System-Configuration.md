# NodeJS Configuration

## What is NodeJS Configuration?

**NodeJS Configuration** allows Jenkins administrators to configure one or more Node.js installations that can be used by Jenkins pipelines and jobs.

Node.js is widely used for developing:

- React applications
- Angular applications
- Vue applications
- Express.js APIs
- NestJS applications
- Frontend build tools

Instead of hardcoding the Node.js installation path in every pipeline, Jenkins provides centralized Node.js management.

---

# Why Do We Need NodeJS Configuration?

Modern web applications require Node.js for:

- Installing packages
- Building frontend applications
- Running unit tests
- Executing npm scripts
- Creating production builds

Without Node.js,

Jenkins cannot build JavaScript-based applications.

# Common Mistakes

## NodeJS Not Installed

Pipeline

```text
node

command not found
```

Install Node.js on the build agent.

---

## Wrong NodeJS Version

Example

```text
React Project

Requires

NodeJS 22

тЖУ

Agent Uses

NodeJS 16

тЖУ

Build Failed
```

Always verify project compatibility.

---

## Different Versions on Different Agents

```text
Agent A

NodeJS 18

--------------------

Agent B

NodeJS 20

--------------------

Agent C

NodeJS 22
```

Different versions may produce different build results.

Standardize Node.js versions.

---

## Hardcoding Node Path

Avoid

```bash
/opt/node/bin/npm install
```

Use Jenkins Tool Configuration instead.

---

# Best Practices

- Configure Node.js globally.
- Use LTS versions in production.
- Standardize Node.js versions across all agents.
- Install Node.js manually in enterprise environments.
- Keep `package-lock.json` under version control.
- Periodically update Node.js after compatibility testing.

---

# Troubleshooting

## Node Command Not Found

Verify

```bash
node -v
```

Install Node.js if missing.

---

## npm Install Failed

Check

- Internet connectivity
- Proxy configuration
- npm registry availability
- Package name

---

## Build Works Locally but Fails in Jenkins

Verify

- Node.js version
- npm version
- Environment variables
- `package-lock.json`

---

## Wrong Tool Loaded

Check

```text
Manage Jenkins

тЖУ

Tools

тЖУ

NodeJS Installations
```

Verify that the configured tool name matches the Jenkinsfile.

---

# Key Points

- NodeJS Configuration allows Jenkins to centrally manage Node.js installations.
- Pipelines reference configured Node.js tool names instead of installation paths.
- Node.js is required for building JavaScript and frontend applications.
- Enterprise environments typically standardize on an LTS version of Node.js across all build agents.
- Proper Node.js configuration ensures consistent and reliable frontend builds in CI/CD pipelines.
