# Common Interview Questions

## What Are GitHub Issues?

### Short Answer

GitHub Issues are used to track bugs, features, tasks, and discussions.

### Detailed Explanation

Issues provide a structured way to manage work and collaborate on development activities.

### Production Example

```text
Bug Found

↓

Issue Created

↓

Assigned

↓

Fixed

↓

Closed
```

### Follow-Up Questions

```text
What Are Labels?

What Are Milestones?

Can Issues Be Assigned?
```

---

## What Are GitHub Projects?

### Short Answer

GitHub Projects are project management boards used to track and organize work.

### Detailed Explanation

Projects provide visibility into progress and allow teams to manage tasks using Kanban-style workflows.

### Production Example

```text
To Do

↓

In Progress

↓

Done
```

### Follow-Up Questions

```text
Can Projects Be Automated?

How Are Projects Different From Issues?

Can Projects Replace Jira?
```

---

## What Is GitHub Wiki?

### Short Answer

GitHub Wiki is a documentation platform integrated with repositories.

### Detailed Explanation

Wikis help teams maintain architecture documents, runbooks, troubleshooting guides, and operational procedures.

### Production Example

```text
EKS Runbook

↓

Stored In Wiki
```

### Follow-Up Questions

```text
Difference Between Wiki And README?

Who Can Edit Wikis?

Can Wikis Be Version Controlled?
```

---

## Difference Between Issues And Projects?

### Short Answer

Issues represent work items, while Projects organize and track those work items.

### Detailed Explanation

Issues are individual tasks. Projects provide a visual workflow for managing multiple issues.

### Production Example

```text
Issue

↓

Task
```

```text
Project

↓

Task Management
```

### Follow-Up Questions

```text
Can Projects Contain Multiple Issues?

Can Issues Exist Without Projects?

How Do Teams Use Both Together?
```

# Key Takeaways

```text
GitHub Issues track bugs, features, and tasks.

Labels help categorize work.

Milestones group related issues.

GitHub Projects provide Kanban-style task management.

Projects improve visibility and planning.

GitHub Wiki stores detailed documentation.

README provides overview documentation.

Wiki provides operational and technical documentation.

Issues, Projects, and Wikis work together to improve team collaboration.
```
