# Docker Logs and Troubleshooting

## Introduction

Running containers is easy.

Troubleshooting containers in production is where DevOps engineers spend most of their time.

Common issues include:

```text
Application Not Starting

Container Exiting Immediately

Port Not Accessible

Memory Issues

Network Problems

Image Pull Failures

Volume Mount Issues
```

---

Docker provides several commands to troubleshoot containers.

```text
docker logs

docker inspect

docker exec

docker stats

docker top

docker events
```

---

# Docker Logs

## What are Docker Logs?

Docker captures:

```text
STDOUT

STDERR
```

generated by containers.

---

View Logs

```bash
docker logs container-id
```

---

Example

```bash
docker logs frontend
```

---

Output

```text
Application Started

Connected To Database

Listening On Port 8080
```

---

# Follow Logs in Real Time

Similar to:

```bash
tail -f
```

---

Command

```bash
docker logs -f frontend
```

---

Useful for:

```text
Application Monitoring

Troubleshooting Startup Issues
```

---

# Show Last N Lines

```bash
docker logs --tail 50 frontend
```

---

Displays:

```text

```

---

# Add Timestamps

```bash

```

---

Useful during:


# Common Interview Questions

## How Do You Troubleshoot a Failed Container?

### Short Answer

Use docker ps -a, docker logs, docker inspect, and docker exec.

### Detailed Explanation

Start by identifying the container state, reviewing logs, inspecting configuration, and validating connectivity.

### Production Example

Catalogue container failing due to MongoDB connection issues.

### Follow-Up Questions

- Which command shows logs?
- How do you access a running container?

---

## What is docker inspect?

### Short Answer

Displays detailed metadata about Docker objects.

### Detailed Explanation

Provides configuration, networking, volume, environment variable, and health-check information.

### Production Example

Finding container IP address and mounted volumes.

### Follow-Up Questions

- How do you inspect networks?
- How do you check mounted volumes?

---

## What is OOMKilled?

### Short Answer

Container terminated because it exceeded memory limits.

### Detailed Explanation

The Linux kernel kills the container process when memory consumption exceeds the configured limit.

### Production Example

Java application exceeding 512 MB memory limit.

### Follow-Up Questions

- How do you verify OOMKilled?
- How can it be prevented?

---

## What does docker stats show?

### Short Answer

Real-time CPU, memory, network, and I/O usage.

### Detailed Explanation

Useful for performance monitoring and resource troubleshooting.

### Production Example

Detecting memory-intensive containers.

### Follow-Up Questions

- How do you identify high memory usage?
- How does this relate to Kubernetes metrics?

---

# Key Takeaways

```text
docker logs is the first command used for troubleshooting.

docker inspect provides complete container metadata.

docker exec allows access to running containers.

docker stats shows real-time resource usage.

docker top displays running processes.

docker events shows Docker activity in real time.

Network issues can be validated using ping and nc.

Volume issues can be checked using docker volume inspect.

OOMKilled indicates memory exhaustion.

A structured troubleshooting approach reduces incident resolution time.
```
